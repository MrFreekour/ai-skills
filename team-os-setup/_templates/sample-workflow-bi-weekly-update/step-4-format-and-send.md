# Step 4 — Format and send

Pruned draft → final output → delivered.

## Render the final markdown

Save as `output/{YYYY-MM-DD}.md`. Use today's date — the file is the canonical record.

Format follows the version you picked in Step 2 (leadership / partner / exec). Keep the structure consistent across updates so the audience knows where to look for what.

## Rendering optional formats

- **Slack-ready summary:** 1–2 paragraphs, no markdown headers, no tables. Drop links inline. Save to `output/{YYYY-MM-DD}-slack.md` (gitignored unless your team checks them in).
- **Email-ready version:** for non-technical leadership. Plain prose, no jargon, no path references. Save to `output/{YYYY-MM-DD}-email.md`.

## Delivery

- Markdown version: link from the team channel + tag relevant reviewers
- Slack version: paste into the channel where leadership lives (typically the cross-functional channel)
- Email version: send to the leadership distribution list

## After delivery

- Note any feedback in `reference/feedback-log.md` — drives the next iteration of this workflow
- If a stakeholder asked something the format didn't address, update the structure for next time
- Don't archive old outputs — they're the team's running record. Only delete if the workflow is being deprecated.

## Output of this step

The final markdown file at `output/{YYYY-MM-DD}.md` + any optional renderings.
